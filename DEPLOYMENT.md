Create TiDB Serverless Cluster\n\n1. Go to [TiDB Cloud Console](https://tidbcloud.com)\n2. Create new Serverless cluster\n3. Configure:\n   - Region: Choose closest to your users\n   - Cluster name: `shri-algo-prod`\n   - Password: Generate strong password\n4. Wait for cluster to initialize (5-10 minutes)\n5. Copy connection string (MySQL format)\n\n### 2. Apply Migrations\n\n```bash\n# Set DATABASE_URL\nexport DATABASE_URL=\"mysql://user:password@host:port/database?ssl={\\\"rejectUnauthorized\\\":true}\"\n\n# Apply migrations\npnpm db:push\n```\n\n## Frontend Deployment (Vercel)\n\n### 1. Prepare Repository\n\n```bash\n# Push code to GitHub\ngit add .\ngit commit -m \"Production ready: Shri Algo v1.0\"\ngit push origin main\n```\n\n### 2. Deploy to Vercel\n\n1. Go to [Vercel Dashboard](https://vercel.com/dashboard)\n2. Click \"Add New Project\"\n3. Select your GitHub repository\n4. Configure:\n   - **Framework**: Vite\n   - **Build Command**: `pnpm build`\n   - **Output Directory**: `dist/public`\n   - **Install Command**: `pnpm install`\n\n5. Set Environment Variables:\n   ```\n   VITE_APP_ID=your-manus-app-id\n   VITE_OAUTH_PORTAL_URL=https://manus.im/oauth\n   VITE_FRONTEND_FORGE_API_URL=https://api.manus.im\n   VITE_FRONTEND_FORGE_API_KEY=your-frontend-api-key\n   VITE_ANALYTICS_ENDPOINT=https://analytics.manus.im\n   VITE_ANALYTICS_WEBSITE_ID=your-website-id\n   ```\n\n6. Click \"Deploy\"\n7. Wait for deployment to complete\n8. Note the Vercel URL (e.g., `https://shri-algo.vercel.app`)\n\n### 3. Configure Backend URL\n\nAfter backend deployment, update frontend environment variable:\n\n```\nVITE_BACKEND_URL=https://shri-algo-backend.onrender.com\n```\n\nRedeploy frontend.\n\n## Backend Deployment (Render)\n\n### 1. Create Web Service\n\n1. Go to [Render Dashboard](https://dashboard.render.com)\n2. Click \"New +\" → \"Web Service\"\n3. Select \"Deploy an existing repository\"\n4. Connect your GitHub repository\n\n### 2. Configure Service\n\n1. **Name**: `shri-algo-backend`\n2. **Runtime**: Node\n3. **Build Command**: `pnpm install && pnpm build`\n4. **Start Command**: `pnpm start`\n5. **Plan**: Free (or paid for production)\n\n### 3. Set Environment Variables\n\nAdd all required environment variables:\n\n```\nNODE_ENV=production\nDATABASE_URL=mysql://user:password@host:port/database?ssl={\"rejectUnauthorized\":true}\nJWT_SECRET=generate-strong-random-string\nVITE_APP_ID=your-manus-app-id\nOAUTH_SERVER_URL=https://api.manus.im\nVITE_OAUTH_PORTAL_URL=https://manus.im/oauth\nOWNER_OPEN_ID=your-owner-open-id\nOWNER_NAME=Your Name\nBUILT_IN_FORGE_API_URL=https://api.manus.im\nBUILT_IN_FORGE_API_KEY=your-forge-api-key\nVITE_FRONTEND_FORGE_API_URL=https://api.manus.im\nVITE_FRONTEND_FORGE_API_KEY=your-frontend-api-key\n```\n\n### 4. Deploy\n\n1. Click \"Create Web Service\"\n2. Wait for deployment (3-5 minutes)\n3. Note the Render URL (e.g., `https://shri-algo-backend.onrender.com`)\n4. Verify API is working:\n   ```bash\n   curl https://shri-algo-backend.onrender.com/api/trpc/auth.me\n   ```\n\n## Custom Domain (Optional)\n\n### Vercel\n\n1. Go to project settings\n2.
